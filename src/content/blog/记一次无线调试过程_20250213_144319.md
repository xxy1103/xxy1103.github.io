此前对android设备进行adb调试时，都是将设备与电脑有线连接，输入adb命令，可是进行想要优化手表时，galaxy watch 并没有插口，要想输入adb命令，只能通过无线调试，所以记录下无线调试的过程。

# 手表连接wifi，打开无线调试

1. 手表与电脑连接同一wifi
2. 手表打开开发者模式（关于手表->软件信息->软件版本(连续点击多次，直到提示开发者模式已打开)）
3. 进入开发者模式，打开adb调试

![1739430119113](image/记一次无线调试过程_20250213_144319/1739430119113.webp)

4. 打开无线调试

   ![1739430148301](image/记一次无线调试过程_20250213_144319/1739430148301.webp)
5. 点击配对新设备，显示如下界面：

   ![1739430575237](image/记一次无线调试过程_20250213_144319/1739430575237.webp)

手表保持此界面就好，接下来就是电脑的工作

# 电脑配对

1. 首先确定电脑装好adb调试工具，确认方法：
   在powershell中输入：

   ```
   adb --version
   ```

![1739430389343](image/记一次无线调试过程_20250213_144319/1739430389343.webp)

2. 确认安装后，我们就可以继续输入：

   ```
   adb pair <手表端显示的IP地址和端口>
   ```

![1739430585610](image/记一次无线调试过程_20250213_144319/1739430585610.webp)

显示successfully 就是配对成功。

3. adb 连接手表：

   ![1739430720449](image/记一次无线调试过程_20250213_144319/1739430720449.webp)

   ```
   adb connnect <ip地址和端口>
   ```

注意，此处的端口号不是之前配对时所用的端口号，而是连接时的端口号。

![1739430832701](image/记一次无线调试过程_20250213_144319/1739430832701.webp)

即为连接成功。

# adb 命令

输入adb命令时，在命令前添加adb shell 

```
adb shell 命令
```
